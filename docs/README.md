## Getting Started

### For New Developers

1. Read the [Setup Guide](../SETUP.md) to get the project running
2. Review [Component Documentation](./COMPONENTS.md) to understand the component library
3. Study [Hooks Documentation](./HOOKS.md) to learn data management patterns
4. Reference [Utilities Documentation](./UTILITIES.md) for helper functions

### For New Users

1. Start with the [User Guide](./USER_GUIDE.md)
2. Focus on your role-specific features (Admin or Counselor)
3. Practice with test data before using in production
4. Refer back to the guide as needed

## Contributing to Documentation

When adding new features, please update the relevant documentation:

- Add new components to `COMPONENTS.md`
- Document new hooks in `HOOKS.md`
- Add utility functions to `UTILITIES.md`
- Update user-facing features in `USER_GUIDE.md`
